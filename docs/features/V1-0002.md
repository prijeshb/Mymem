## Backend

### Wiki RAG Chunking (`mymem/rag/wiki_chunker.py`) — NEW

Markdown/header + parent-child chunking strategy for wiki pages:

- **Step 1 — frontmatter strip:** Extract `title`, `domain`, `tags` from YAML frontmatter
- **Step 2 — heading split:** `MarkdownHeaderTextSplitter` on `#` / `##` / `###` → sections (parent chunks, max 4096 chars)
- **Step 3 — child split:** `RecursiveCharacterTextSplitter` splits each section into ~300-token child chunks with 30-token overlap
- **Embed text:** `"{page_title} > {heading_path}: {child_text}"` — prefix improves embedding precision
- **Stored per chunk:** `source_path`, `source_slug`, `heading_path`, `parent_text`, `chunk_type`, `page_title`, `domain`, `tags`
- Never raises — returns `[]` on missing file, empty body, or missing dependency

### RAG Store (`mymem/rag/store.py`)

- `delete_source()` — removes all chunks for a source path (needed for re-indexing on page update)
- Schema extended with wiki-specific nullable fields: `heading_path`, `parent_text`, `chunk_type`, `page_title`, `domain`, `tags`

### RAG Ingest (`mymem/rag/ingest.py`)

- `ingest_wiki_page()` — orchestrates `chunk_wiki_page → embed → delete_source (if force) → insert_chunks`
- `force=True` clears old chunks before re-inserting (used on every write to keep index current)

### Pipeline Ingest (`mymem/pipeline/ingest.py`)

- `_rag_index_wiki()` — fire-and-forget async function called via `asyncio.ensure_future` after every wiki page write
- Never blocks ingest — best-effort, logs warnings on failure

### Dependencies (`pyproject.toml`)

- `langchain-text-splitters` added (provides `MarkdownHeaderTextSplitter` + `RecursiveCharacterTextSplitter`)

## Frontend

### Dashboard Layout (`DashboardPage.tsx`, `App.tsx`)

Complete layout refactor from 2-column grid to a full-height 3-zone layout:

**Layout structure:**
```
┌──────────────┬───────────────────────────┬──────────────────────┐
│ Left (240px) │     Center (flex-1)        │  Right panel (420px) │
│              │                            │  [slide-in, toggled] │
│  Overview    │  [Wiki Pages btn]          │                      │
│  Stats       │                            │  Wiki Pages list     │
│              │  Quick Ask card            │  Sticky header       │
│  Domain      │  ├─ input + domain + Ask   │  Scrollable rows     │
│  Heatmap     │  ├─ save checkbox          │  Pagination footer   │
│              │  └─ answer (55vh scroll)   │                      │
└──────────────┴───────────────────────────┴──────────────────────┘
```

**Key decisions:**
- Right panel: `w-0` → `w-[420px]` via `transition-all duration-300` (Claude Code panel pattern). Inner div fixed at `w-[420px]` so content doesn't reflow during animation.
- Wiki Pages toggle: standalone button above the Quick Ask card, `flex justify-end`
- Answer output: `max-h-[55vh] overflow-y-auto` — capped, scrollable, never pushes page
- `max-w-screen-2xl` on `<main>` — fills wider viewports (was `max-w-7xl`)
- `h-[calc(100vh-56px-2rem)]` on dashboard root — full-height panel layout, no page scroll
